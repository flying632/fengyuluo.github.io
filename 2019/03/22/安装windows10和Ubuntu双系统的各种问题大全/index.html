安装windows10和Ubuntu双系统踩坑

发表于 2019-03-22 | 分类于 踩坑集

[TOC]

安装详情

Ubuntu版本的选择

​ Ubuntu的版本，最初选择的是Ubuntu kylin版本，中国的衍生版本，装完后各种莫名bug，搜了一下解决办法，都是推荐重装原生Ubuntu系统，就重装了原生版本。

Ubuntu 18.04.2 LTS 版本

LTS版本，Long Term Support 长时间支持版本（三年），一般的版本支持为18个月

详细的安装步骤

  1. 制作U盘启动盘

    使用UltralSO制作启动盘，如果下载后的镜像打开后，发现只有一个EFI文件夹，恭喜中奖了，你下的UltrISO版本太低了，换个最新的版本的就好了，直接去官网下载，然后使用即可

    打开软碟通，点击左上角的“文件”——“打开”，然后选择ubuntu镜像存放的位置，确定。

    点击上方菜单的“启动”——“写入硬盘映像”。

    选择写入方式USB-HDD+，隐藏启动分区选择无，然后点击写入，就成功的把ubuntu镜像写入u盘里了，最后显示刻录成功，这样ubuntu的u盘启动盘就制好了。

  2. 创建磁盘分区

    从windows的系统磁盘管理中分出Ubuntu的空间，大小根据自己的需要来定

    （1）右击我的电脑，管理，选择磁盘管理

    ​

    ​

    （2）选择剩余空间较大的可分配磁盘，右键并选择“压缩卷”，这里选择压缩E盘50G左右的空间：

    ​

    （3）点击“压缩”之后，E盘后部出现黑色的50G“未分配空间”，磁盘分区过程完成。

  3. 禁用快速启动

    按住Win + X(请记住这个万能的组合)，选择“电源选项”，”选择电源按钮的功能”，”更改当前不可用的设置”，取消选择”启用快速启动(推荐)”。

  4. 开始安装，重启电脑，U盘启动

    HP是按F10，进入BIOS界面，在启动方式中将U盘启动调为优先级最高，

    在这里BIOS界面有个可选的操作Secure boot （安全启动），建议选为disable（禁用）

  5. 进入后按部就班的点击继续即可，进入分区界面，选择“其他选项”，“继续”

    然后就对我们预先空出来的50G的空闲空间进行操作

    分区这样分：

    / ：16G （16384MB）主分区 exe4

    swap：16G （16384MB）逻辑分区，交换分区

    /boot ：1G （1024MB）

    efi ：2G （2048MB）efi系统分区

    /home ：剩余的空间都放着

  6. 之后就正常安装重启即可

安装时需要注意的点

  • ​ /boot的挂载空间大一些，为好，不然后面会出问题，系统内核一更新，空间就不够了，类似这种

  • 上面的分区方式是针对当前的Windows系统是基于EFI启动的，而不是 legacy boot方式，所以需要eft系统分区，并且也不需要使用EasyBCD这样的工具添加引导

问题总结

Ubuntu安装失败，如何删除

进入windows 10系统，下载MBRfix软件

MBRfix软件是一款可以修复系统主引导记录数据的小软件

如果是64位的win10系统，则需要运行MbrFix64.exe

运行MbrFix64.exe程序

  1. 使用CMD命令运行命令控制台

  2. 将当期路径切换到MbrFix64.exe文件所在的的路径

  3. 运行如下命令：MbrFix64.exe /drive 0 fixmbr

  4. 按照命令提示输入：Y

如果提示失败，权限不够，以管理员的身份运行，没有错误信息出现则成功了

之后就可以在Windows的磁盘管理中将Ubuntu占用的分区直接删除掉了，编程空闲空间

Windows环境如何删除efi分区

  1. Win+R打开运行，输入Diskpart打开来进行分区管理。

    输入list disk 查看当前磁盘列表，然后通过select disk X 来选择想要操作的磁盘，选中磁盘后，输入list partition来查看该磁盘下的分区：

  2. select partition X来选定EFI分区

  3. delete partition override

Ubuntu重启或关机都会黑屏，只有左上角的一个光标

1、通过之前安装Ubuntu的U盘，USB启动系统，然后点击“Try Ubuntu”

2、添加Ubuntu的boot-repair的PPA源：

sudo add-apt-repository ppa:yannubuntu/boot-repair && sudo apt-get update

3、安装boot-repair：

sudo apt-get install -y boot-repair

4、在命令行中输入boot-repair，选择第一项

5、会出现很多命令，一条条copy进去，执行

6、修复成功后重启系统，问题解决。
